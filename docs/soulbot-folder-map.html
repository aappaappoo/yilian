Soulmeet / Aini 项目结构说明

Soulbot 文件夹与文件用途地图

这份文档把仓库拆成前端、后端、共享包、角色素材、部署配置和本地运行数据几层来说明。 源码文件逐项列出用途；第三方依赖、Git 数据、缓存、日志和虚拟环境按目录规则归类，避免把外部生成内容误认为业务设计。

当前仓库是动漫陪伴 AI 产品，不是企业后台。UI 设计应保持柔和、梦幻、清爽、玻璃质感。

Vue + Vite FastAPI LiveKit Soul companion
apps/web前端入口，首页、聊天页、状态管理、实时语音连接和卡片化回复。
apps/backendFastAPI 后端，文本/语音会话、记忆、鉴权、角色素材和工具调用。
packagesworkspace 共享包，承载 i18n、类型、UI 和 3D 角色渲染。
plugins确定性工具插件，提供天气、车票、本地搜索、旅行规划和联网资料读取。

整体链路

从用户输入到回复展示，项目的大致运行路径如下。

Web 输入apps/web 负责聊天输入、语音控制、角色展示和卡片渲染。
API 层apps/backend/main.py 注册文本、WebRTC、鉴权、对话、天气等路由。
会话层runtime_session.py 管理会话、历史、流式输出、中断和记忆同步。
Agent/工具core/soul_companion 调 LLM 工具循环或关键词回退路由。
结果展示前端将文本、任务进度、天气、车票、旅行 artifact 转成柔和卡片。
顶层目录与文件 后端文件 前端文件 共享包 角色与知识 部署配置 运行数据 维护注意

维护注意

敏感文件： .env、deploy/.env、certs/*.pem、deploy/certs/*.pem、日志、记忆文件、插件配置里的第三方服务配置，都应按敏感信息处理。
开发优先阅读路径： 前端聊天先读 apps/web/src/pages/chat.vue、components/chat/DesktopChatLayout.vue、components/chat/MobileChatLayout.vue； 后端聊天先读 apps/backend/core/conversation/runtime_session.py、apps/backend/core/soul_companion/runtime.py、apps/backend/core/soul_companion/agent_loop.py； 实时语音先读 apps/backend/core/api/webrtc.py 和 apps/web/src/composables/useWebRTC.ts。
设计约束： 聊天 UI 改动应保留 Aini 角色、语音控制、天气控制、侧栏、输入框和对话详情面板；风格保持柔和、梦幻、圆润、玻璃质感，移动端不能横向溢出。
生成位置：docs/soulbot-folder-map.html。可直接用浏览器打开，也可作为项目交接文档维护。
