# Scenic Query Templates

本文件用于构造景点 `local_search` 查询。

## 基本参数

- `place`：destination，或更具体的商圈/车站/地标。
- `category`：`scenic`。
- `keyword`：优先用具体 POI、地标或短关键词。不要把天气、老人偏好、少走路、打车方便等筛选词全部塞进 `keyword`。

## 模板

天气不好 + 老人少走路：
优先查具体室内 POI，例如 `科技馆`、`博物馆`、`展馆`、`中山路`。

天气不好 + 厦门 + 老人一日游：
首查 `厦门市博物馆 厦门科技馆 文化馆`；如果不可用，再查 `中山路`。

天气适合户外 + 看海 + 老人：
可查 `鹭江道`、`环岛路观光巴士`、`白城沙滩` 等短关键词，再筛选少走路方案。

普通老人友好：
查 `观光巴士`、`老街`、`公园` 等短关键词，再筛选少走路方案。

名胜古迹偏好：
查具体古迹或短关键词，例如 `南普陀寺`、`古迹`。

## 查询纪律

- 不要在天气返回前查景点。
- 不要用过于抽象的城市级关键词反复试探，也不要用过长的复合关键词。
- 如果已有景点结果中有可用于行程的候选，优先筛选，不要继续补查。
- 只有结果为空、全部明显不适合天气、全部明显需要长距离步行/爬坡/排队，或区域明显不合适时，才补查景点。
- 如果具体 POI 查询返回了可用结果，即使结果数量少，也足够进入餐饮查询和最终行程，不要为了凑满列表继续补查。
