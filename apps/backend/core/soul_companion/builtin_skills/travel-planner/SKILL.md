# Travel Planner Skill

你负责把实时、细致的旅游规划拆成多个原子工具查询，再综合成适合老人出行的中文行程建议。

## 什么时候使用

使用本 skill：
- 用户明确要今天、明天、具体日期的一日游或多日游精细行程规划。
- 用户明确要老人、父母、长辈、少走路、轻松不累的具体行程安排。
- 用户明确给出出发地和目的地，要求跨城出行规划。
- 用户同一句话组合提到天气、车票、景点、美食、住宿中的两个或更多信息。
- 用户表达目的地游玩偏好，例如看海、名胜古迹、少走路、预算低、不吃海鲜、清淡饮食、室内游玩，并要求具体行程安排。

不要使用本 skill：
- 用户只是泛泛询问旅游攻略、怎么玩、几日游、自由行或第一次去某地，且没有要求实时信息或精细排程时，使用 `guide`。
- 用户只查天气时，只调用 `weather`。
- 用户只查火车票、动车票、高铁车次或余票时，只调用 `train_tickets`。
- 用户只查多少钱、票价或某个席别价格时，只调用 `train_ticket_price`。
- 用户只查美食、住宿或景点时，只调用 `local_search`。
- 用户只是设置提醒、查询健康数据或闲聊时，不参与。

## 槽位和默认值

从用户问题和上下文提取：
- destination：目的地城市、区域或地标。
- origin：出发地。只有用户给出出发地，或明确要查车票/跨城交通时才需要。
- date：出行日期；未给出时，车票和精细行程默认按“明天”。
- days：旅行天数；未给出时默认 1 天。
- preferences：看海、少走路、室内、便宜、清淡、不吃海鲜等偏好。
- budget：预算描述。

如果没有 destination，先问用户想去哪里。
如果用户只是说目的地游玩偏好但没给 origin，不要追问出发地，继续做目的地游玩规划。
如果用户要查车票或“从哪里到目的地”的交通，但缺 origin，先问“您从哪里出发？”。

## 渐进披露

先按当前任务加载必要 reference，不要一次性加载无关内容。多个 reference 互不依赖时，可以同一轮并行调用 `skill_view(name="travel-planner", file_path=...)`。

- `references/dependency-flow.md`：旅游工具依赖图。开始执行旅游规划前必须加载。
- `references/elder-low-walk.md`：老人、少走路、轻松节奏、午休和打车友好策略。用户提到老人、父母、长辈、少走路、轻松、不累时必须加载。
- `references/weather-routing.md`：雨天、高温、大风、台风、暴晒等天气降级策略。天气返回后如需要更细的降级策略再加载。
- `references/scenic-query-templates.md`：景点 `local_search` 查询模板。无法确定景点关键词或首查失败时再加载。
- `references/food-query-templates.md`：美食 `local_search` 查询模板。无法确定餐饮关键词或首查失败时再加载。

## 核心执行原则

- 不要调用聚合旅行工具，只调用原子工具。
- 互不依赖的工具同一轮并行调用；依赖前一步结果的工具必须串行。
- 天气、去程车次、返程车次互不依赖，参数足够时第一批并行。
- 景点依赖天气，天气返回后再查。
- 美食依赖景点或主要活动区域，景点/区域确定后再查。
- 一日游返程默认查 `latest`，查到后从已有返程结果中筛选；不要再把同一路线同一天改成 `earliest` 重查。
- 住宿只在多日游、住一晚或用户明确提到酒店/住宿时查。
- `train_tickets` 是可选工具：只有 origin 存在，或用户明确问车票/动车/高铁/跨城交通时使用。
- `train_ticket_price` 是可选工具：只有用户明确询问“多少钱、票价、价格、二等座/一等座价格”时使用。
- 补查必须有明确理由：工具失败、结果为空、结果与天气/老人少走路/饮食偏好冲突、区域明显错误，或用户追加新条件。已有结果足够支撑答复时直接收尾。

## 最终答复要求

最终答复必须基于工具结果，不要编造未查到的天气、车次、价格、地点、营业状态。

用简洁中文组织：
- 先给明确结论和整体节奏。
- 再按“交通、天气、景点、吃饭、住宿、老人友好提醒”整理。
- 每天最多 1-2 个重点地点，中午安排休息，避免连续爬坡、长队排队和太晚返程。
- 如果某类工具失败或没查到，说明没查到，并给下一步建议。
- 说明来源：天气/景点/美食/住宿来自高德地图，车次来自 12306。
- 提醒余票、价格、营业时间、酒店房态以官方平台或商家实时结果为准；未调用 `train_ticket_price` 时不要给出具体票价。
